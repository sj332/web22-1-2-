5장 폼 활용하기



  1. <form>요소

    2. <form>요소의 사용

    • 회원가입, 상품구매, 키워드 검색 등 사용자로부터 정보를 받을 때
    • 사용자와 애플리케이션이 상호작용
    • 사용자 입력 ⇒ 전송버튼 ⇒ 애플리케이션에 전달 ⇒ 실행결과 반환

    <form>요소의 역할

    • 사용자가 입력하는 정보를 하나로 묶어서 애플리케이션에 전달
    • 다양한 입력 양식을 그룹핑하고 전송방법을 설정
    • <form> 요소 내 사용자의 정보 입력 양식
    • <input>, <textarea>, <select> 등의 입력 요소를 이용

    <form>요소
    요소 이름속성속성 설명
    <form>요소 method 속성 데이터 전송 방식 지정 get방식
    post방식
    action 속성데이터 처리할 애플리케이션 주소 (웹서버 경우 URL)
    name속성폼 요소에 대한 이름

    method 속성

    method속성
    get방식post방식
    전송할 데이터를 URL 주소 뒤에 포함프로그램의 입출력 방식을 사용
    간단한 데이터에는 편리, 그러나 보안에 취약데이터 양에 제한 없다

    action 속성

    데이터 처리할 애플리케이션 주소 (웹서버 경우 URL)

    name 속성

    폼 요소에 대한 이름 애플리케이션에 전달될 변수명

  2. <input> 요소

    1. type 속성: 입력 폼의 유형
      • 텍스트 관련: text, password
      • 선택 관련 : radio, checkbox
      • 버튼 : submit, reset, button, image
      • 기타 : file, hidden
    2. name 속성(문자열 입력 필드, 한 줄)
      • 애플리케이션에 전달될 변수명
    3. value 속성(문자열 입력 필드, 한 줄)
      • value 속성 : 입력 받은 값을 전달 (입력창 초기문자)
    4. min , max(날짜) 속성
    5. <input>요소에 추가된 새로운 입력 키워드
      • 텍스트 관련: email, URL, tel, search
      • 날짜와 시간 : date, month, week, time, datetime, datetime-local
      • 색상 및 숫자: number, range, color

    type 속성

    암호 입력 필드 "password"
    보안이 필요한 문자 입력, ‘•’ 으로 표시 (더 이상 보호 기능 없음)
    라디오 버튼, "radio"
    name 속성값이 같은 그룹에서 value 값을 하나만 선택하여 전달
    예, “member=yes&sex=male”의 형태로 전달
    체크박스 선택, "checkbox"
    체크박스에 표시된 모든 value 속성값 들이 애플리케이션으로 전송
    전송 버튼, "submit"
    입력 데이터 전달
    <form> 요소에 있는 모든 입력 데이터가 <form>의 action 속성에서 지정한 애플리케이션 프로그램으로 전송
    초기화 버튼, "reset"
    현재 입력된 양식을 모두 지움
    일반 버튼, "button"
    다양한 용도로 사용
    이미지 버튼, "image"
    alt 대체 문자열 사용 가능
    파일 선택하기, "file"
    파일 업로드 가능
    데이터 숨김, "hidden"
    시스템에서 특정 데이터를 처리하고 싶을 때
    예: 사용자 IP 주소

    추가된 서식이 있는 type의 키워드

    이메일 주소입력, "email"
    이메일 주소가 존재 하는지 까지는 확인 불가 이나, 형식이 맞게 작성되었는지는 확인

    URL 주소입력, "URL"
    인터넷 주소 표기 규정에 맞게 “http://” 가 추가 되었는지 확인해 줌
    또는 value 속성으로 미리 “http://” 를 지정할 수도 있다

    검색창 입력, "search"
    검색어를 입력하면 오른쪽에 “X”가 표시되어 입력한 검색어를 취소할 수 있음

    날짜 입력(일), "date"
    연-월-일

    날짜 입력(월), "month"
    연-월

    날짜 입력(주), "week"
    연-주 (번호순서 1월 첫 주가 ‘W01’)

    "위/아래” 버튼으로 시간 조정, "time"

    UTC 국제표준 시간대, "datetime"

    현지 시간, "datetime-local"

    value 속성

    입력 받은 값을 전달 (입력창 초기문자)

    pattern 속성

    전화번호 입력
    • 원하는 전화번호와 자리수에 유효한 패턴을 지정
    • 정규 표현식으로 정의, 자바스크립트에서 사용하는 패턴과 동일
    • pattern=”[A-Za-z]{no}” , pattern=”[0-9]{no}” , pattern=”[0-9]+” , pattern=”[A-Za-z0-9]{min, max}”

    placeholder 속성

    입력할 자리수 표시 (단순히 표시만)

    min 속성, max 속성



  3. <textarea>요소

    4. 텍스트 영역 필드(여러줄)
    표시 영역보다 많은 텍스트를 입력하면 스크롤바 <textarea name=”이름” cols=”열의 수” rows=”행의 수”> </textarea>


  4. <select>요소

  5. <select>요소 내에 <option>항목

    • 드롭다운 형태(size= 1), 혹은 스크롤 박스 형태(size > 1)
    • multiple 속성 : 다중 선택
    • 각 항목은 <select> 요소 내에서 <option> 요소로 정의
    select요소 내의 <option>요소

  6. <label >요소

    7. 텍스트 라벨: 특정 입력 필드에 연결
    • 데이터 전달에는 영향이 없음
    • 버튼이외의 텍스트영역을 선택해도 선택이 가능
    • <label for=”입력아이디”>


  7. <fieldset>요소

    8. 그룹핑
    <fieldset>요소
    • 폼 양식을 그룹핑하는 범위를 지정
    • 사용자의 시각적 편의를 위해서 제공
    • 입력 필드의 그룹 주위에 기본 스타일로 테두리 선
    		name 속성 : 그룹의 이름을 지정하는 속성
    form 속성 : 폼과 연결
    disabled 속성 : 그룹 내의 모든 하위 입력 요소들을 비활성화

  8. <legend>요소

    • 그룹을 구분하기 위한 그룹의 제목 라벨
    • <fieldset> 요소에 포함되는 첫 번째 자식 요소로서 한번만 사용
    • 그룹 라벨의 스타일은 그룹을 구분하는 선의 중간에 걸쳐서 표시
